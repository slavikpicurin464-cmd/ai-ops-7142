Этот скил говорит про артефакты по смыслу — таблица гипотез крео, план запуска и распределения бюджета, таблица аналитики. Если работаешь по методичке курса AI-таргетолог — эти артефакты соответствуют Листам 1-10. Если работаешь без курса — используй любую таблицу с теми же столбцами.

# Winner Variations

Ты разворачиваешь победившее крео в 5-8 вариаций и 1-2 диких эксперимента. Цель - выжать максимум из найденного выигрышного угла и параллельно подготовить замены до того как победитель устанет.

## Когда использовать

Срабатывает на этапе масштабирования и решений (Этап 10 в курсе) ПОСЛЕ закрытия этапа аналитики недели (Этап 9 в курсе) с решением SCALE по конкретному адсету. Это вход в следующий цикл - что запускаем в неделю N+1.

Также вызывайся когда ученик прямо просит:
- «вариации победителя по адсету X»
- «масштабирую крео X - дай 5 вариаций»
- «дикие эксперименты под победителя»
- «scale креатива»

НЕ подгружай если:
- По адсету нет решения SCALE (только KILL/HOLD) - для них вариации не делаются.
- Этап аналитики недели (Этап 9 в курсе) не закрыт - сначала диагностика, потом масштаб.
- Это первая итерация (нет факта по победителю) - сначала запускаем 12 гипотез через `creative-brief-writer`.
- Победитель отработал больше 21 дня без обновления - это уже не масштаб, это замена, нужен новый цикл этапов офферов, подходов и производства крео (Этапы 5-7 в курсе).

## Что делает

Шаг 1. Принимает входы.

- Имя адсета-победителя (например `INGLISH_CARM_PAIN_UGC_v1`).
- Фактический сигнал победителя. Для видео и лидгена это обычно CPL + hook rate / hold rate / CTR. Для товарной статики и e-com сигналом победителя могут быть ROAS, CPP (cost per purchase), CTR, доля заказов в рассрочку - бери ту метрику, по которой адсет реально выиграл, не натягивай CPL туда где продажа.
- Исходный Magnific-промпт (12 колонок таблицы гипотез крео — Лист 7 в курсе) - сегмент, подход, формат, hook, body, CTA, актёр, длительность, фон, BGM, версия, статус.
- Опционально - правило 10% бюджета на дикие эксперименты (если в плане запуска и распределения бюджета — Лист 8 в курсе — настроено).

Шаг 2. Применяет правило одного параметра.

Меняешь ОДИН параметр за раз. Это нужно для чистой атрибуции - чтобы понять что именно влияет на метрику.

Какой набор параметров брать - смотри на формат победителя. Победитель видео - берёшь видео-набор. Победитель товарная статика (карточка, баннер, packshot) - берёшь набор для статики. Не смешивай наборы.

Набор для ВИДЕО - 5 параметров:

- hook - первые 3 секунды (звук, текст, визуал).
- actor - кто говорит (другой возраст / другой пол / другая локация / голос за кадром).
- format - UGC → Cinematic → Static → Reel → Carousel.
- duration - 6 / 15 / 30 / 60 секунд.
- bg - фон / локация съёмки (офис / дом / на улице / минимализм).

Набор для товарной СТАТИКИ - 5 параметров:

- ракурс/кроп - как снят товар (фронт / три четверти / деталь крупно / товар в руках / на модели).
- фон/подложка - белый / цветной / фактура / lifestyle-сцена / градиент.
- подача цены и рассрочки - есть / нет плашки, цена крупно / мелко, «0-0-12» вынесено вперёд / спрятано.
- модель / манекен / flat-lay - кто или что носит товар (живая модель / манекен / только товар / flat-lay раскладка).
- один товар / комплект - показываем одну позицию или собранный комплект / образ.

Набор для REELS-ИМПУЛЬСА / ECOM_IMPULSE - 5 параметров:

- hook - первые 1-2 секунды (визуал, текст-плашка, action-кадр, не «говорящая голова»).
- demo-angle - что и как показываем (action - товар в действии / reaction - реакция на товар / before-after - до и после / price-reveal - акцент на цене).
- trend-sound - звук с актуального тренда vs нейтральный фоновый vs голос-войсовер.
- price-display - плашка с ценой крупно / плашка мелко / без плашки совсем.
- CTA - свайп вверх / «в шапке профиля» / прямой клик в Shop / без явного CTA (товарный клик через тег).

В наборе ECOM_IMPULSE специально убраны actor и duration как оси.
- Actor не имеет смысла - на 15-секундном demo нет «говорящей головы», есть товар и руки / тело без лица.
- Duration не имеет смысла - Reels-импульс работает в коридоре 8-15 сек, шире не идёт, и тестировать 30 vs 60 = тестировать нерелевантные варианты.

Параметр «язык» - отдельная ось вариации для двуязычных рынков (RU / KZ / смешанный). Работает и для видео, и для статики, и для Reels-импульса. Это дешёвый и логичный A/B - текст в кадре и озвучка меняются, всё остальное держится как у победителя.

Не меняй два параметра одновременно даже если очень хочется. Иначе через 7 дней не поймёшь что сработало.

Шаг 3. Генерирует 5-8 вариаций по схеме нейминга.

`{Адсет-победитель}.v1.1-hook` / `.v1.2-actor` / `.v1.3-format` / `.v1.4-duration` / `.v1.5-bg`.

Можно делать по 2 вариации на параметр (v1.1a, v1.1b) если есть две сильные гипотезы внутри одного параметра. Тогда выйдет 8 строк.

Шаг 4. В колонке Body text каждой вариации пишет гипотезу почему меняем этот параметр.

- v1.1-hook → «исходный hook „Понимаю каждое второе, улыбаюсь как дурочка" даёт hook rate 34%. Гипотеза - hook „Завтра созвон на английском - ты к нему готова?" может поднять до 40%+ потому что добавляет срочность».
- v1.2-actor → «исходный актёр - карьеристка 32 Алматы. Гипотеза - актёр 28 лет Минск (другой Кейс на той же боли) даст лучше выгорание по аудитории Минск».
- и так далее.

Шаг 5. Опционально - 1-2 диких эксперимента.

Дикие эксперименты идут отдельно от 5-8 вариаций и не считаются как SCALE. Бюджет на них - 10% от расхода на победителя.

Дикие идеи для ВИДЕО-победителя:
- Длинный формат 60-120 секунд (vs 15-30 на победителе).
- Мем-формат с трендовым звуком.
- UGC от нетипичного для сегмента лица (мужчина-карьерист в женском сегменте, бабушка-предпринимательница).
- Странный hook - тишина первые 3 секунды, обратный отсчёт, чёрный экран с текстом.
- Формат без человеческого лица - анимация / motion graphics / только text на цветном фоне.

Дикие идеи для СТАТИКИ-победителя:
- 3D-рендер товара вместо фотосъёмки.
- Карусель-комплект - не один товар, а собранный образ / набор по слайдам.
- Инфографика размерной сетки или сравнения - вместо чистого packshot.

Каждый дикий маркируется `.exp1`, `.exp2`. Если зайдёт - переходит в обычные вариации в следующем цикле. Если не зайдёт - выкидываем без сожаления, на то и эксперимент.

Шаг 6. Финальная таблица - 12 колонок Magnific формата + версия + гипотеза. Идёт прямо в таблицу гипотез крео (Лист 7 в курсе) следующего цикла.

## Формат ответа

Один. Таблица вариаций с явной колонкой «Что меняем» - один параметр на строку.

Четыре. Финальное распределение бюджета на неделю - в строках, с процентами.

Пять. Если просят 5 вариаций - давай 5, не 8. Не натягивай объём.

## Quality-gate

Перед выдачей проверь:

- В каждой вариации меняется РОВНО ОДИН параметр относительно победителя. Если изменилось два - переписывай. «Язык» (RU / KZ / смешанный) считается валидным одним параметром - менять язык, держа всё остальное как у победителя, это корректная вариация.
- Гипотеза в Body text каждой вариации опирается на конкретную метрику победителя, не на «мне кажется». Для видео это hook rate / CTR / frequency, для товарной статики и e-com - ROAS / CPP / CTR / доля заказов в рассрочку. Метрика должна совпадать с той, по которой адсет реально выиграл.
- Имена вариаций в формате `{адсет}.v1.{N}-{параметр}` - читаемо в Ads Manager.
- Если адсет в HOLD или KILL по этапу аналитики недели (Этап 9 в курсе) - вариации не выдаёшь, отправляешь ученика обратно за решением SCALE.
- Дикие эксперименты явно помечены как `.exp` и идут отдельно от обычных вариаций.

CTR - диагностический сигнал, не критерий выбора победителя. Победитель определяется по cost per result (CPL/CPP для лидгена; ROAS/CPP для e-com), не по CTR. Гипотезы для вариаций строятся вокруг ключевой метрики, по которой адсет выиграл. Hook rate / hold rate используем для понимания почему крео цепляет, но решение SCALE - по деньгам.

## Ротация крео - альтернативная стратегия для широких ниш

Для широких ниш (INFOBIZ массовые курсы, ECOM с большим каталогом) кроме классических вариаций одного победителя работает стратегия «ротация пачками». Неделя крутим 10 крео, неделя - другие 10. Возвращаем первую пачку через неделю-две.

Зачем это делать.

- Снимает усталость аудитории. Frequency на широкой нише накручивается медленно, но всё равно накручивается. Смена пачки = пауза в показе одного и того же лица / hook.
- Даёт алгоритму second chance на крео которое не зацепилось в первый раз. FB-аукцион в момент первого запуска мог попасть в неудачный сегмент аудитории, dropping → дешёвый clicker pool. Через неделю перезапуск ловит другую часть аудитории и иногда крео «оживает».
- Защищает от ситуации «слили 10 крео за 2 недели, теперь нечего показывать».

Когда применять.

- Широкая аудитория - минимум 2-3 млн в гео, не одна узкая боль.
- Каталог продуктов или массовый инфопродукт (не один курс с одним сегментом).
- Спенд позволяет держать 2 пачки по 10 крео в готовности.

Когда НЕ применять. Узкие ниши - локальный бизнес в одном городе, узкий B2B-сегмент, нишевой продукт на маленькую аудиторию. Там перезапуски бесполезны: аудитория конечная, через неделю те же люди увидят тот же крео и реакция не изменится. Для узких ниш работает классическая логика - искать новых победителей через A/B одного параметра, не ротация пачек.

## Пример - InglishGo, победитель Karierist_Pain_UGC_v1

Контекст. Адсет `INGLISH_CARM_PAIN_UGC_v1` отработал 7 дней. Спенд 280 USD. CPL 6.36 USD vs план 8. Hook rate 34%. Hold rate 28%. CTR 1.9%. Frequency 1.8. По этапу аналитики недели (Этап 9 в курсе) - решение SCALE.

Исходный промпт. Сегмент CARM. Подход «Боль». Формат UGC. Hook «Понимаю каждое второе слово на встрече, улыбаюсь как дурочка». Body «8 недель × 3 урока по 60 минут с международным преподавателем. 1200+ выпускниц уже работают в зарубежных компаниях». CTA «Записаться на пробное за 0 USD». Актёр - женщина 32 Алматы, talking head. Длит - 22 сек. Фон - домашний офис. BGM - нет.

Вариации.

| Версия | Что меняем | Hook | Гипотеза |
|---|---|---|---|
| v1.1-hook | hook | «Завтра созвон на английском - ты к нему готова?» | hook rate 34% уже сильный, но добавление срочности (цитата Лист 2 #10) может поднять до 40%+ |
| v1.1b-hook | hook | «Через 8 недель ты будешь говорить с американцами и не уставать через час» | hook rate упор на result а не pain - тест на симметричную гипотезу |
| v1.2-actor | actor | (как у победителя) | Актёр - женщина 29 Минск (другая страна сегмента). Гипотеза - локальная аутентичность поднимет CTR на BY-аудитории |
| v1.3-format | format | (как у победителя) | Формат Cinematic 15 сек вместо UGC 22 сек. Гипотеза - короткий cinematic может зайти лучше для холодной аудитории второй итерации |
| v1.4-duration | duration | (как у победителя) | Длительность 12 сек (сокращённая версия). Гипотеза - hold rate 28% выше среднего, но цена за просмотр упадёт при 12 сек |
| v1.5-bg | bg | (как у победителя) | Фон - офис open space с коллегами на фоне (vs домашний офис). Гипотеза - усиление контекста «коллеги слушают» поднимет hook rate |

Дикие эксперименты.

| Версия | Концепт | Гипотеза | Риск |
|---|---|---|---|
| exp1 | Тишина первые 3 сек, текст «Молчишь на созвоне? Это пройдёт за 8 недель» появляется на чёрном экране | Тишина в ленте Reels = pattern interrupt. Может поднять hook rate до 45% или провалить до 15% | 30-40 USD бюджета на 4 дня теста |
| exp2 | 60-секундный mini-документальный формат с 3 выпускницами «до/после» | Длинный формат на холодной = риск, но если зайдёт - LTV-канал на много недель | 50-70 USD на 5 дней |

Бюджет на неделю.
- Победитель v1 - 50% (140 USD/неделю, продолжаем держать).
- Вариации v1.1-v1.5 - 40% (по 22 USD на каждую = 6 строк).
- Дикие exp1+exp2 - 10% (по 14 USD).
- Итого - 280 USD/неделю (равно прошлой неделе, без увеличения).

## Чего не делаешь

- Не меняешь два параметра одновременно «потому что вместе должно лучше». Атрибуция по параметру тогда невозможна.
- Не делаешь больше 8 вариаций - алгоритм не оптимизирует распылённое.
- Не убираешь самого победителя из ротации при запуске вариаций. Победитель держит 40-60% бюджета пока новые не подтвердят перевес.
- Не считаешь дикие эксперименты «вариацией». Они отдельная категория с правилом 10% бюджета.
- Не делаешь вариации без указания конкретного факта победителя (hook rate, CTR). Без данных это не масштабирование, это слепая генерация.

## Пример - InglishGo, победитель Relocant_Srochnost_Video_v1

Контекст. Адсет `INGLISH_RELO_SROCH_VIDEO_v1` отработал 9 дней. Спенд 320 USD. CPL 7.50 USD vs план 8. Hook rate 31%. Hold rate 22%. CTR 1.6%. Frequency 2.1. По этапу аналитики недели (Этап 9 в курсе) - решение SCALE.

Исходный промпт. Сегмент RELO. Подход «Срочность» (через страх не успеть). Формат Video Cinematic. Hook «Билеты в Ереван куплены, английский в нуле». Body «47 релокантов прошли наш курс за 8 недель до переезда. 3 урока по 60 минут с международным преподавателем». CTA «Не опоздай - старт группы 03.06». Актёр - женщина 28, voice-over. Длит - 18 сек. ## Пример - Smart Pet, победитель SMARTPET_PREMCORM_STATIC_v1

Контекст. Интернет-магазин кормов и аксессуаров для животных «Smart Pet». Адсет `SMARTPET_PREMCORM_STATIC_v1` - карусель «Премиум-корма Acana и Orijen -15%». Отработал 7 дней. Спенд 250 USD. ROAS 4.1 vs план 3.0. CPP 8.20 USD. CTR 2.4%. Add-to-cart 6.2%. Доля заказов в рассрочку 38%. По этапу аналитики недели (Этап 9 в курсе) - решение SCALE. Победитель - товарная статика, не видео, поэтому берём набор для статики, не для видео.

Исходный промпт. Сегмент - владельцы крупных пород 30-45. Подход «Выгода» (скидка на премиум-сегмент). Формат Static Carousel - 5 слайдов. Слайд 1 - packshot Acana на белом фоне, цена крупно. Слайд 2 - Orijen. Слайды 3-5 - другие SKU из премиум-линейки. Подача цены - старая цена зачёркнута + новая + плашка «-15%». Рассрочка вынесена отдельной строкой «0-0-12».

## Когда не работаешь

При первичной сборке 12 гипотез крео - это `creative-brief-writer`.

При сборке матрицы 4 подходов на новый сегмент - это `schwartz-podhody`.

При диагностике почему адсет не вышел в SCALE - это `campaign-diagnoser`.

При прогоне новых офферов через светофор - это `offer-generator` (этап светофор-проверки встроен).

При написании готовых текстов крео - это `ru-copywriter`.

## Типичные ошибки ученика - пасхалки

«Один креатив тестирую - не цепляет - отключаю». Один крео не даёт алгоритму выбора. Минимум 3 концепта параллельно + 2-3 вариации на каждый параметр. Иначе FB не на чём оптимизироваться - аукцион просто крутит то что есть в одну дешёвую аудиторию, и ты делаешь вывод «крео плохой» из шума. Нет вариативности - нет данных - нет решения.

«CTR победителя 1.8%, у вариации 0.6% - вариация плохая, кил». Не торопись. Если cost per result (CPL / CPP / ROAS) у вариации в пределах нормы - она работает. CTR может быть низким потому что FB цепляет другое (более дорогое или более узкое) ядро ЦА - меньше людей кликают, но те кто кликает - покупают. Решение по деньгам, не по CTR. Сначала смотри сколько стоит результат, потом - почему он столько стоит.
