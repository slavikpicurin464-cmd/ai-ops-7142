# Reality Check Metrics

## Когда использовать

Срабатывает в двух точках протокола:

1. На Этапе 1 ПОСЛЕ заполнения Лист 1 и прохождения Quality-Gate. Бот делает реальность-чек автоматически - не спрашивает, не предлагает. Это барьер от провального запуска по фантастическим вводным.

2. На Этапе 8, день 5-7 после старта кампаний. Реальность-чек по факту - сверка фактического CPL с планом и расчётным предельным. Если расхождение более 2× - мини-откат на Этап 1 для пересчёта экономики до продолжения теста.

Также вызывайся когда ученик прямо просит «посчитай предельный CPL для клиента X», «реалистичен ли план Y USD CPL для ниши Z», «потянем ли мы план клиента на 8 USD CPL».

НЕ подгружай если реальность-чек уже сделан и метрики не менялись.

## Что делает

Шаг 1. Берёт из Лист 1 - средний чек, маржу, конверсию лид→продажа (CR), целевой ROAS, целевой CPL клиента, профиль ниши.

Шаг 2. Считает по формулам:
- Предельный CAC (cost per acquisition customer) = Средний чек × Маржа.
- Предельный CPL = (Средний чек × Маржа × CR лид→продажа) / Целевой ROAS.

Если CR не указан - берёт бенчмарк по нише, помечает буквой B (бенчмарк):
- INFOBIZ онлайн-обучение - 10-15%.
- LOCAL_SERVICE - 25-40% (короткая воронка).
- ECOM - 2-5% (импульсная покупка).
- B2B_SAAS - 5-10% (длинная воронка).
- HIGH_TICKET - 3-8% (созвоны).
- PREMIUM_TRAVEL / LONG_CYCLE_HIGH_TICKET - 12-22% (длинный цикл с менеджером, премиум-туризм, недвижимость, элитные услуги, B2B-консалтинг от 2000 USD).
- SOFT_EXPERT (психологи, медицина, юристы-консультанты) - 8-18% (отложенное решение из-за деликатности темы).

Если ROAS не указан - 3 для INFOBIZ/LOCAL_SERVICE, 2 для ECOM, 4 для B2B_SAAS, 5 для HIGH_TICKET, 3 для SOFT_EXPERT, 2-3 для PREMIUM_TRAVEL (часто с расчётом на LTV повторных поездок).

Шаг 3. Берёт реалистичный стартовый CPL по бенчмарку рынка СНГ для профиля:
- INFOBIZ образование - 12-25 USD.
- LOCAL_SERVICE - 5-15 USD.
- ECOM малый/средний чек - 3-10 USD.
- B2B_SAAS - 30-80 USD (квалифицированный лид).
- HIGH_TICKET - 50-200 USD.
- PREMIUM_TRAVEL / LONG_CYCLE_HIGH_TICKET - 40-120 USD (длинный цикл, чек 2000+ USD, менеджер закрывает по телефону).
- SOFT_EXPERT - 8-20 USD (онлайн-эксперты health/mental/legal).

Важно. Эти диапазоны - стартовые для холодной аудитории на холодных кампаниях. После 4-6 недель оптимизации реальный CPL может быть в 1.5-2 раза ниже. Для ниш с длинным циклом покупки бенчмарк стартового CPL смещён вверх потому что цикл скрывает часть конверсий за окно атрибуции.

Связка с campaign-diagnoser. Если на Этапе 8 факт CPL > 2× план - ДО пересчёта экономики прогони `campaign-diagnoser` узел Атрибуция. Если CAPI/домены/UTM не настроены - фактический CPL завышен на 20-50% против реального. После починки атрибуции пересчитай и только потом меняй план с клиентом.

Шаг 4. Сравнивает план клиента vs предельный расчётный vs реалистичный стартовый. Выдаёт вердикт по 3 сценариям:

Сценарий A. План ≥ предельный И в коридоре реалистичного стартового. Вердикт «план реалистичный». Идём дальше без задержек.

Сценарий B. План ≤ предельный, но НИЖЕ нижней границы реалистичного стартового. Вердикт «план на грани, на старте будем выше». Три варианта решения - снизить ROAS-цель до 1.5-2 на первый цикл; поднять чек или маржу; заложить работу над воронкой (лендинг + лид→продажа CR). Ученик идёт с клиентом ДО ЗАПУСКА.

Сценарий C. План ВЫШЕ предельного. Вердикт «план математически невозможен, экономика не сходится». Запуск не имеет смысла без пересборки. Бот блокирует переход на Этап 2 до возврата клиента с пересчётом.

Шаг 5. Фиксирует в Лист 1 две новые строки - «Предельный CPL расчётный» и «Реалистичный стартовый CPL по рынку». В колонке Значение - числа. В пометках - сценарий A/B/C.

## Промпт-шаблон

```
Реальность-чек метрик по клиенту {имя}.

Вводные из Лист 1.
- Профиль ниши - {INFOBIZ / LOCAL_SERVICE / ECOM / B2B_SAAS / HIGH_TICKET}.
- Средний чек - {Y} USD.
- Маржа - {Z}% ({факт / бенчмарк}).
- CR лид→продажа - {W}% ({факт / бенчмарк нишы}).
- Целевой ROAS клиента - {R}.
- Целевой CPL клиента - {X} USD.

Считаю.
Предельный CAC = {Y} × {Z}% = {CAC} USD.
Предельный CPL = {CAC} × {W}% / {R} = {лимит} USD.

План клиента {X} USD vs предельный {лимит} USD = {в плюс / в минус на сколько}.

Реалистичный стартовый CPL для ниши {профиль} в СНГ - {диапазон} USD.

План клиента {X} USD vs реалистичный диапазон = {в коридоре / ниже нижней / выше верхней}.

Вердикт - сценарий {A / B / C}.

{Если B} - план на грани. До запуска обсуди с клиентом три варианта:
1. Снижаем ROAS-цель до 1.5-2 на первый цикл, потом дотягиваем.
2. Поднимаем чек или маржу если есть резерв.
3. Закладываем работу над лендингом и отделом продаж до CR {целевой}%.

{Если C} - план математически невозможен. Запуск блокируется до пересчёта. Возвращайся с обновлёнными вводными.

Строки для Лист 1.
- «Предельный CPL расчётный - {лимит} USD».
- «Реалистичный стартовый CPL по рынку - {диапазон} USD».
- «Реальность-чек - сценарий {A/B/C}».
```

## Примеры

Пример 1. Сценарий A - инфобиз реалистичный план.
Вход. INFOBIZ. Чек 280 USD. Маржа 70%. CR 12% (B - бенчмарк ниши). ROAS план 3. CPL план 8 USD.
Расчёт. Предельный CAC = 280 × 0.7 = 196 USD. Предельный CPL = 196 × 0.12 / 3 = 7.84 USD.
Реалистичный стартовый CPL для INFOBIZ - 12-25 USD.
Вердикт. План 8 USD ≥ предельный 7.84 USD (в плюс на 2%). НО план 8 USD НИЖЕ нижней границы реалистичного 12 USD - это сценарий B, не A.
Решение. На грани. На старте будем 12-18 USD, не 8 USD. До запуска обсуди с клиентом - либо снижаем ROAS-цель до 2 (тогда предельный CPL поднимается до 11.76), либо поднимаем чек до 380 USD, либо закладываем 4-6 недель на оптимизацию лендинга для повышения CR до 18-20%.

Пример 2. Сценарий C - локалка нереалистичный план.
Вход. LOCAL_SERVICE (стоматология). Чек 500 USD. Маржа 40%. CR 30% (B). ROAS план 5. CPL план 4 USD.
Расчёт. Предельный CAC = 500 × 0.4 = 200 USD. Предельный CPL = 200 × 0.3 / 5 = 12 USD.
Реалистичный стартовый CPL для LOCAL_SERVICE - 5-15 USD.
Вердикт. План 4 USD ≤ предельный 12 USD - технически OK. Но план 4 USD НИЖЕ нижней границы реалистичного 5 USD - сценарий B на самом нижнем краю.
Решение. На грани. Реалистичный CPL первой итерации 7-12 USD, не 4. До запуска обсуди - либо ROAS-цель 3 вместо 5 (тогда предельный CPL поднимается до 20), либо больший бюджет на тест чтобы быстрее выйти на нормальные цифры.

Пример 3. Сценарий C - ECOM невозможный план.
Вход. ECOM (косметика). Чек 30 USD. Маржа 40%. CR 3% (B). ROAS план 4. CPL план 1 USD.
Расчёт. Предельный CAC = 30 × 0.4 = 12 USD. Предельный CPL = 12 × 0.03 / 4 = 0.09 USD.
Вердикт. План 1 USD ВЫШЕ предельного 0.09 USD в 11 раз - сценарий C. Экономика не складывается даже по верхним планкам.
Решение. Запуск блокируется. К клиенту с цифрами. Варианты - (1) поднять AOV через апселл/комплекты до 60-80 USD; (2) увеличить маржу через производителя; (3) снизить ROAS-цель до 1.5 как первая итерация на сборе данных. Без пересчёта не запускаемся.

## Edge cases

Когда НЕ применять.
- Реальность-чек уже сделан на Этапе 1 и метрики не менялись. Не дублируем.
- У ученика нет данных для расчёта (не знает маржу, чек, CR, ROAS). Сначала идём в стандартный Этап 1 опросник, заполняем Лист 1. Потом реальность-чек.

Когда применять с оговорками.
- ROAS не указан клиентом - берём бенчмарк по нише, помечаем буквой B. Если фактический ROAS клиента сильно отличается от бенчмарка (например клиент с подпиской и LTV 1200 USD имеет реальный ROAS 5-8) - расчёт получится консервативный, реальная экономика лучше. Помечаем «бенчмарк может быть занижен».
- CR указан клиентом, но «по ощущениям» - помечаем B (бенчмарк) пока не подтверждено выгрузкой из CRM. На Этапе 9 проверяем факт vs ожидание.
- Чек указан с большим диапазоном (например «150-800 USD») - берём средневзвешенный или нижнюю границу как консервативный сценарий. Помечаем «расчёт по нижней границе чека - реальный AOV может улучшить экономику».
- Бизнес с recurring revenue (подписка) - в расчёт идёт не чек первой покупки, а LTV за 6-12 месяцев. Если LTV не известно - помечаем «LTV TBD, расчёт по чеку первой покупки, реальная экономика после 6 месяцев будет лучше».

Когда применять второй раз (на Этапе 8).
- День 5-7 после старта кампаний. Берём средневзвешенный CPL за 3-7 день (первые 2 дня learning phase не считаем).
- Если факт ≤ 2× план - реальность-чек в норме, идём в Этап 9 по графику.
- Если факт > 2× план И спенд набрал 30-50% правила 50 конверсий - мини-откат. Возвращаем на Этап 1 для пересчёта ДО продолжения теста. К клиенту с цифрами и решением.
- Если на Этапе 1 стоял флаг risk-CPL (сценарий B) и факт совпадает с риск-прогнозом - сразу разворачиваем план Б который был согласован до запуска.
