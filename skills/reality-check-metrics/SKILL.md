# Reality Check Metrics

## Когда использовать

Срабатывает в двух точках протокола:

1. На Этапе 1 ПОСЛЕ заполнения Лист 1 и прохождения Quality-Gate. Бот делает реальность-чек автоматически - не спрашивает, не предлагает. Это барьер от провального запуска по фантастическим вводным.

2. На Этапе 8, день 5-7 после старта кампаний. Реальность-чек по факту - сверка фактического CPL с планом и расчётным предельным. Если расхождение более 2× - мини-откат на Этап 1 для пересчёта экономики до продолжения теста.

Также вызывайся когда ученик прямо просит «посчитай предельный CPL для клиента X», «реалистичен ли план Y USD CPL для ниши Z», «потянем ли мы план клиента на 8 USD CPL».

НЕ подгружай если реальность-чек уже сделан и метрики не менялись.

## Что делает

Шаг 1. Берёт из Лист 1 - средний чек, маржу, конверсию лид→продажа (CR), целевой ROAS, целевой CPL клиента, профиль ниши.

Шаг 2. Считает по формулам:
- Предельный CAC (cost per acquisition customer) = Средний чек × Маржа.
- Предельный CPL = (Средний чек × Маржа × CR лид→продажа) / Целевой ROAS.

Оговорка для REAL_ESTATE. Маржа застройщика в процентах низкая (15-25%), но в абсолюте огромная - 15% от сделки на 200к USD это 30к USD. Считать предельный CPL надо от абсолютной маржи на сделку, не от процента. И разведи две метрики: CPL за сырой лид и cost per booked appointment за визит в офис. Предельная стоимость визита = абсолютная маржа на сделку × CR визит→договор / целевой ROAS. Сырой лид без доведения до визита экономику не считает.

Если CR не указан - берёт бенчмарк по нише, помечает буквой B (бенчмарк):
- ИНФОБИЗ / INFOBIZ онлайн-обучение - 10-15%.
- ЛОКАЛЬНЫЙ-СЕРВИС / LOCAL_SERVICE - 25-40% (короткая воронка).
- ИНТЕРНЕТ-МАГАЗИН / ECOM - 2-5% (импульсная покупка).
- B2B-СЕРВИС / B2B_SAAS - 5-10% (длинная воронка).
- ПРЕМИУМ / HIGH_TICKET - 12-22% (длинный цикл с менеджером, премиум-туризм, элитные услуги, high-end коучинг, B2B-консалтинг от 2000 USD).
- МЯГКИЙ-ЭКСПЕРТ / SOFT_EXPERT (психологи, медицина, юристы-консультанты) - 8-18% (отложенное решение из-за деликатности темы).
- REAL_ESTATE / REAL_ESTATE - CR лид→бронирование 8-15%, CR лид→договор 3-7%, маржа застройщика 15-25%. Пометь B - требует подтверждения выгрузкой из CRM.
- RELIGIOUS_TRAVEL / RELIGIOUS_TRAVEL - CR лид→продажа 6-12% [предположение]. Пометь B - требует подтверждения выгрузкой из CRM.

Если ROAS не указан - 3 для INFOBIZ/LOCAL_SERVICE, 2 для ECOM, 4 для B2B_SAAS, 5 для HIGH_TICKET, 3 для SOFT_EXPERT. Для HIGH_TICKET с премиум-туризмом цель часто 2-3 (расчёт на LTV повторных поездок). Для REAL_ESTATE - ROAS-цель 4-8, считается на горизонте 6-12 месяцев, не на окне атрибуции. Для RELIGIOUS_TRAVEL - 2-2.5 без LTV-допущения о повторах, умра не регулярная покупка.

## Предельная цена по ШАГАМ воронки

Для длинной воронки одного «предельного CPL» мало - ученик не понимает где экономика ломается по ступеням. Считай предельную цену каждого шага отдельно.

Формула. Предельная цена шага = предельная цена клиента × произведение CR всех ступеней между этим шагом и покупкой (обратный пересчёт от продажи к верху воронки).

Worked example - маммопластика. Средний чек 4100 USD × маржа 50% = 2050 USD предельная цена клиента (= сколько можно заплатить за саму покупку чтобы выйти в ноль).

Воронка с CR по ступеням:
- подписчик → профиль - 50%
- профиль → клик - 20%
- клик → просмотр сайта - 80%
- просмотр → лид - 10%
- лид → переписка - 80%
- переписка → звонок - 40%
- звонок → покупка - 20%

Таблица предельных цен по шагам:
- подписчик в инсте - 41 USD
- посещение профиля - 82 USD
- клик по ссылке - 205 USD
- просмотр сайта - 256 USD
- лид (оставил контакт) - 2050 USD
- переписка - 2562 USD
- звонок - 6400 USD

Правило. Всё что фактически ниже предельной цены шага - в плюс. Идеал - х2 окупаемость, фактическая цена шага в два раза ниже предельной.

Зачем считать по шагам. Когда CPL формально в плюсе (например лид 150 USD при предельном 2050), но звонок по факту стоит 8000 USD - экономика ломается на ступенях лид→переписка→звонок, не в рекламе. Без разложения по шагам ученик чинит не то место - крутит крео когда надо чинить отдел продаж.

Шаг 3. Берёт реалистичный стартовый CPL по бенчмарку рынка СНГ для профиля:
- ИНФОБИЗ / INFOBIZ образование - 12-25 USD.
- ЛОКАЛЬНЫЙ-СЕРВИС / LOCAL_SERVICE - 5-15 USD.
- ИНТЕРНЕТ-МАГАЗИН / ECOM малый/средний чек - 3-10 USD.
- B2B-СЕРВИС / B2B_SAAS - 30-80 USD (квалифицированный лид).
- ПРЕМИУМ / HIGH_TICKET - 50-200 USD (длинный цикл, высокий чек, менеджер закрывает по телефону).
- МЯГКИЙ-ЭКСПЕРТ / SOFT_EXPERT - 8-20 USD (онлайн-эксперты health/mental/legal).
- REAL_ESTATE / REAL_ESTATE - 30-100 USD за сырой лид. Пометь B - требует подтверждения выгрузкой из CRM.
- RELIGIOUS_TRAVEL / RELIGIOUS_TRAVEL - 30-90 USD [предположение]. Пометь B - требует подтверждения выгрузкой из CRM.
- ПОДПИСЧИК В ИНСТЕ (СНГ 2026) - 1-2 USD норма. Пометь B. Старый миф «0.01-0.07 USD» устарел - его до сих пор тащат в курсах, но он не отражает текущие аукционы Meta. Считаем от предельной цены по экономике (см. раздел «Предельная цена по ШАГАМ воронки»), не от мифа.

Важно. Эти диапазоны - стартовые для холодной аудитории на холодных кампаниях. После 4-6 недель оптимизации реальный CPL может быть в 1.5-2 раза ниже. Для ниш с длинным циклом покупки бенчмарк стартового CPL смещён вверх потому что цикл скрывает часть конверсий за окно атрибуции.

Множитель сезонности. Для ниш с выраженным сезонным пиком (RELIGIOUS_TRAVEL, часть ECOM) давай вилку «в сезон / не в сезон» отдельными числами, а не один усреднённый диапазон. Запуск вне сезона - отдельная пометка в вердикте: стартовый CPL выше, поток лидов ниже, выводы по экономике делать осторожно.

Связка с campaign-diagnoser. Если на Этапе 8 факт CPL > 2× план - ДО пересчёта экономики прогони `campaign-diagnoser` узел Атрибуция. Если CAPI/домены/UTM не настроены - фактический CPL завышен на 20-50% против реального. После починки атрибуции пересчитай и только потом меняй план с клиентом.

Двухступенчатая воронка для SOFT_EXPERT. Если KPI клиента - бесплатный созвон (не оплата) - считай ДВА предельных CPL отдельно. CPL_лид (на созвон-знакомство) и CPL_клиент (на оплату). CPL_лид = CPL_клиент × CR(созвон→оплата). Пример: CR созвон→оплата 35%, CPL_клиент 16 USD → CPL_лид 5.6 USD на оплату или 16 USD на созвон. Сравнивай план клиента с метрикой ИЗ ТОЙ ТОЧКИ воронки, которую он называет KPI.

Бенчмарки для SOFT_EXPERT на холодной аудитории с осторожной подачей (wellness, похудение, mental health). Нормы ниже базовых - CTR 0.5-1% это норма, не «крео сломан». Hook rate 15-22%, Hold rate 6-10%. Параллельно ставится флаг «осторожная подача» - порог CR на лид-магнит выше (5-10%, не 1-3%), потому что вход бесплатный.

Шаг 4. Сравнивает план клиента vs предельный расчётный vs реалистичный стартовый. Выдаёт вердикт по 3 сценариям:

Сценарий A. План ≥ предельный И в коридоре реалистичного стартового CPL (не ниже нижней границы). Вердикт «план реалистичный». Идём дальше без задержек. Если план ≥ предельного, но НИЖЕ нижней границы реалистичного - это всегда сценарий B, не A. Не путай «экономика сходится» с «по рынку достижимо на старте».

Сценарий B. План ≤ предельный, но НИЖЕ нижней границы реалистичного стартового. Вердикт «план на грани, на старте будем выше». Три варианта решения - снизить ROAS-цель до 1.5-2 на первый цикл; поднять чек или маржу; заложить работу над воронкой (лендинг + лид→продажа CR). Ученик идёт с клиентом ДО ЗАПУСКА.

Сценарий C. План ВЫШЕ предельного. Вердикт «план математически невозможен, экономика не сходится». Запуск не имеет смысла без пересборки. Бот блокирует переход на Этап 2 до возврата клиента с пересчётом.

Шаг 5. Фиксирует в Лист 1 две новые строки - «Предельный CPL расчётный» и «Реалистичный стартовый CPL по рынку». В колонке Значение - числа. В пометках - сценарий A/B/C.

## Промпт-шаблон

```
Реальность-чек метрик по клиенту {имя}.

Вводные из Лист 1.
- Профиль ниши - {INFOBIZ / LOCAL_SERVICE / ECOM / B2B_SAAS / HIGH_TICKET}.
- Средний чек - {Y} USD.
- Маржа - {Z}% ({факт / бенчмарк}).
- CR лид→продажа - {W}% ({факт / бенчмарк нишы}).
- Целевой ROAS клиента - {R}.
- Целевой CPL клиента - {X} USD.

Считаю.
Предельный CAC = {Y} × {Z}% = {CAC} USD.
Предельный CPL = {CAC} × {W}% / {R} = {лимит} USD.

План клиента {X} USD vs предельный {лимит} USD = {в плюс / в минус на сколько}.

Реалистичный стартовый CPL для ниши {профиль} в СНГ - {диапазон} USD.

План клиента {X} USD vs реалистичный диапазон = {в коридоре / ниже нижней / выше верхней}.

Вердикт - сценарий {A / B / C}.

{Если B} - план на грани. До запуска обсуди с клиентом три варианта:
1. Снижаем ROAS-цель до 1.5-2 на первый цикл, потом дотягиваем.
2. Поднимаем чек или маржу если есть резерв.
3. Закладываем работу над лендингом и отделом продаж до CR {целевой}%.

{Если C} - план математически невозможен. Запуск блокируется до пересчёта. Возвращайся с обновлёнными вводными.

Строки для Лист 1.
- «Предельный CPL расчётный - {лимит} USD».
- «Реалистичный стартовый CPL по рынку - {диапазон} USD».
- «Реальность-чек - сценарий {A/B/C}».
```

## Примеры

Пример 1. Сценарий A - инфобиз реалистичный план.
Вход. INFOBIZ. Чек 280 USD. Маржа 70%. CR 12% (B - бенчмарк ниши). ROAS план 3. CPL план 8 USD.
Расчёт. Предельный CAC = 280 × 0.7 = 196 USD. Предельный CPL = 196 × 0.12 / 3 = 7.84 USD.
Реалистичный стартовый CPL для INFOBIZ - 12-25 USD.
Вердикт. План 8 USD ≥ предельный 7.84 USD (в плюс на 2%). НО план 8 USD НИЖЕ нижней границы реалистичного 12 USD - это сценарий B, не A.
Решение. На грани. На старте будем 12-18 USD, не 8 USD. До запуска обсуди с клиентом - либо снижаем ROAS-цель до 2 (тогда предельный CPL поднимается до 11.76), либо поднимаем чек до 380 USD, либо закладываем 4-6 недель на оптимизацию лендинга для повышения CR до 18-20%.

Пример 2. Сценарий C - локалка нереалистичный план.
Вход. LOCAL_SERVICE (стоматология). Чек 500 USD. Маржа 40%. CR 30% (B). ROAS план 5. CPL план 4 USD.
Расчёт. Предельный CAC = 500 × 0.4 = 200 USD. Предельный CPL = 200 × 0.3 / 5 = 12 USD.
Реалистичный стартовый CPL для LOCAL_SERVICE - 5-15 USD.
Вердикт. План 4 USD ≤ предельный 12 USD - технически OK. Но план 4 USD НИЖЕ нижней границы реалистичного 5 USD - сценарий B на самом нижнем краю.
Решение. На грани. Реалистичный CPL первой итерации 7-12 USD, не 4. До запуска обсуди - либо ROAS-цель 3 вместо 5 (тогда предельный CPL поднимается до 20), либо больший бюджет на тест чтобы быстрее выйти на нормальные цифры.

Пример 3. Сценарий C - ECOM невозможный план.
Вход. ECOM (косметика). Чек 30 USD. Маржа 40%. CR 3% (B). ROAS план 4. CPL план 1 USD.
Расчёт. Предельный CAC = 30 × 0.4 = 12 USD. Предельный CPL = 12 × 0.03 / 4 = 0.09 USD.
Вердикт. План 1 USD ВЫШЕ предельного 0.09 USD в 11 раз - сценарий C. Экономика не складывается даже по верхним планкам.
Решение. Запуск блокируется. К клиенту с цифрами. Варианты - (1) поднять AOV через апселл/комплекты до 60-80 USD; (2) увеличить маржу через производителя; (3) снизить ROAS-цель до 1.5 как первая итерация на сборе данных. Без пересчёта не запускаемся.

Пример 4. SOFT_EXPERT - расчёт на LTV пакета, не на чек первой сессии.
Вход. SOFT_EXPERT (психолог). Чек первой сессии 60 USD. Конверсия первой сессии в пакет 50% (B - бенчмарк). Средний пакет 270 USD. ROAS план 3. CR созвон→оплата 35% (B). KPI клиента - бесплатный созвон.
Расчёт. Средневзвешенный LTV первого цикла = 60 + 0.5 × 210 = 165 USD. Предельный CAC = 165 × 0.8 = 132 USD (берём 80% от LTV как запас). Предельный CPL_клиент = 132 × 0.35 / 3 = 15.4 USD. Предельный CPL_лид (на созвон) = 15.4 × 0.35 = 5.4 USD.
Вердикт. Без учёта LTV пакета расчёт по чеку 60 USD даёт предельный CPL = 60 × 0.8 × 0.35 × 0.35 / 3 = 1.96 USD - ложно-низкий, заблокирует жизнеспособный запуск как «сценарий C». С учётом LTV предельный CPL_клиент 15.4 USD укладывается в реалистичный диапазон 8-20 USD для SOFT_EXPERT.
Решение. На старте сравнивай план клиента с CPL_лид (если KPI - созвон) или CPL_клиент (если KPI - оплата). Не смешивай две точки воронки в один CPL.

## Edge cases

Когда НЕ применять.
- Реальность-чек уже сделан на Этапе 1 и метрики не менялись. Не дублируем.
- У ученика нет данных для расчёта (не знает маржу, чек, CR, ROAS). Сначала идём в стандартный Этап 1 опросник, заполняем Лист 1. Потом реальность-чек.

Когда применять с оговорками.
- ROAS не указан клиентом - берём бенчмарк по нише, помечаем буквой B. Если фактический ROAS клиента сильно отличается от бенчмарка (например клиент с подпиской и LTV 1200 USD имеет реальный ROAS 5-8) - расчёт получится консервативный, реальная экономика лучше. Помечаем «бенчмарк может быть занижен».
- CR указан клиентом, но «по ощущениям» - помечаем B (бенчмарк) пока не подтверждено выгрузкой из CRM. На Этапе 9 проверяем факт vs ожидание.
- Чек указан с большим диапазоном (например «150-800 USD») - берём средневзвешенный или нижнюю границу как консервативный сценарий. Помечаем «расчёт по нижней границе чека - реальный AOV может улучшить экономику».
- Бизнес с recurring revenue (подписка) - в расчёт идёт не чек первой покупки, а LTV за 6-12 месяцев. Если LTV не известно - помечаем «LTV TBD, расчёт по чеку первой покупки, реальная экономика после 6 месяцев будет лучше».

Когда применять второй раз (на Этапе 8).
- День 5-7 после старта кампаний. Берём средневзвешенный CPL за 3-7 день (первые 2 дня learning phase не считаем).
- Если факт ≤ 2× план - реальность-чек в норме, идём в Этап 9 по графику.
- Если факт > 2× план И спенд набрал 30-50% правила 50 конверсий - мини-откат. Возвращаем на Этап 1 для пересчёта ДО продолжения теста. К клиенту с цифрами и решением.
- Если на Этапе 1 стоял флаг risk-CPL (сценарий B) и факт совпадает с риск-прогнозом - сразу разворачиваем план Б который был согласован до запуска.
- Для цикла больше 120 дней (REAL_ESTATE, RELIGIOUS_TRAVEL) реальность-чек на Этапе 8 проводим не на день 5-7, а на 4-6 неделе и по leading-метрикам (CPL, cost per visit). Сделки в окно атрибуции не попадут - по ROAS не паникуем раньше 2-3 месяцев. Ранний вердикт по сырому CPL и стоимости визита, не по выручке.

Двуязычные кампании (RU/KZ).
- Закладывай период набора данных дольше и стартовый CPL выше. Бюджет и оптимизация делятся на два языковых сегмента - каждый набирает 50 конверсий медленнее, чем один общий. В вердикте отдельная пометка про удлинённое окно.

Порог бюджета для SOFT_EXPERT.
- Для SOFT_EXPERT с CPL 12-20 USD минимальный жизнеспособный бюджет - 1500-2000 USD/мес чтобы Meta набрала learning phase. При бюджете <1500 USD/мес помечай сценарий B+ (план математически OK, но данных будет мало - оптимизация на 8-12 недель вместо 4-6). К клиенту с цифрами и решением до запуска - либо поднимаем бюджет, либо закладываем удлинённый цикл оптимизации.

## Типичные ошибки ученика - пасхалки

Сюда выноси частые провалы в голове - ученик приходит с ними как с фактом, а внутри них зашита системная ошибка.

«Открутил 10-20 USD, получил 3 лида, продаж 0 - останавливаю кампанию».
- Выборка мала, решение преждевременное. Минимум для оценки - 50 конверсий на адсет ИЛИ 1500-3000 показов на крео. До этой выборки KILL/SCALE не принимай - алгоритм ещё не выбрал кому показывать, продажи могут прийти на следующих десятках лидов. Цифра «3 лида без покупок» статистически ничего не значит - это шум, не сигнал.

«Подписчик стоит 3 USD, а в курсе говорят должен быть 0.5 - паника, что-то сломано».
- Миф устарел. Норма для СНГ 2026 - 1-2 USD за подписчика в инсте, 3 USD это верхняя граница нормы, не катастрофа. Считай не от мифа из курса, а от своей предельной цены подписчика по экономике (см. раздел «Предельная цена по ШАГАМ воронки»). Если предельная 41 USD, а факт 3 USD - ты в плюсе в 13 раз, паниковать не о чем.
