Этот скил говорит про артефакты по смыслу — карта смыслов, таблица офферов, таблица гипотез крео. Если работаешь по методичке курса AI-таргетолог — эти артефакты соответствуют Листам 4, 5, 7. Если работаешь без курса — используй любые таблицы с теми же столбцами.

═══════════════════════════════════════════════════
СТРУКТУРА БРИФА ДИЗАЙНЕРУ / ПРОДАКШНУ
═══════════════════════════════════════════════════

Каждый бриф — таблица с обязательными полями:

| Поле | Что заполнить |
|---|---|
| Клиент / проект | Имя |
| Профиль ниши | Из 8 канонических |
| Бюджетный режим | LITE / STANDARD / PRO |
| Гео | Список стран/городов |
| Сегмент аудитории | Конкретно (не «все», а «новичок-активный в дизайне 22-30 лет») |
| Суб-боль / желание | Из карты смыслов с цитатой |
| Уровень Schwartz | Unaware / Problem-aware / Solution-aware / Product-aware / Most-aware |
| Формат крео | Статика-HTML / UGC-видео / Скринкаст / Карусель / Reels |
| Заголовок крео | Короткая фраза для крупного шрифта |
| Подзаголовок | 1-2 предложения с акцентом |
| Лид-магнит (название) | Короткое название (то что в крео) |
| Лид-магнит (контент) | Описание что внутри, формат файла, объём — для производства |
| СТА | Сильный, с триггер-словом |
| Цветовая схема | 2-3 цвета (бренд + акценты) |
| Шрифт | Sans-serif обычно (Inter, Manrope, Onest) |
| Референсы | Ссылки на 2-3 примера в похожем стиле |
| Стадия воронки | Холод / Тёплый / Ретаргет |
| Запреты | Никаких named-конкурентов, никаких overpromising |

ВАЖНО: лид-магнит = ДВА поля. В крео идёт КОРОТКОЕ название («10 шаблонов»). В блоке «контент» — подробное описание для производства («10 Figma-файлов под разные ниши: B2B / e-com / lifestyle / SaaS / стартап. Можно копировать и заменять контент. Объём — 10 макетов по 5-8 экранов каждый»).

## Когда использовать

Срабатывает на этапе передачи задачи исполнителю — дизайнеру, копирайтеру, монтажёру, продакшн-команде. После того как офферы выбраны (Лист 5 в курсе), сегменты определены (Лист 3-4), формат крео решён (Лист 7).

Триггеры:
- «напиши бриф дизайнеру»
- «сделай ТЗ на UGC»
- «бриф продакшну»
- «бриф копирайтеру на текст крео»
- «бриф монтажёру»
- «creative brief на новую пачку»

НЕ подгружай если:
- Сегмент не определён (сначала research-structurer и client-profile).
- Офферы не сформулированы (сначала offer-generator).
- Уровень Schwartz не выбран (сначала schwartz-podhody).

## Что делает

Шаг 1. Принимает входы.
- Имя клиента и продукт.
- Профиль ниши и бюджетный режим (из client-profile).
- Сегмент аудитории с цитатой боли / желания из карты смыслов (Лист 4).
- Оффер (из Листа 5).
- Уровень Schwartz и подход.
- Формат крео (статика / UGC / скринкаст / карусель / Reels).
- Стиль клиента (из brand-extractor или style-guide-extractor).

Шаг 2. Собирает бриф в таблицу по обязательным полям выше.

Шаг 3. Проверяет на полноту:
- Все 18 полей заполнены или явно помечены «не применимо».
- Лид-магнит описан в ДВУХ полях (название + контент).
- Сегмент описан конкретно, не «все 25-45».
- Цитата из карты смыслов — реальная, не выдуманная.
- Референсы даны ссылками, не описанием.

Шаг 4. Выдаёт бриф готовым к отправке исполнителю.

## Формат вывода

Markdown-таблица в одном блоке. Без воды, без дополнительных пояснений. Исполнитель должен открыть бриф и сразу начать работу.

Если в брифе несколько крео для одной пачки — таблица повторяется N раз, каждое крео отдельной таблицей с заголовком «Крео #N».

## Quality-gate

Перед выдачей проверь:
- Каждое поле имеет конкретное значение или явное «не применимо».
- Сегмент — не абстрактный («женщины 25-45»), а с триггер-ситуацией.
- Заголовок крео — короткий, идёт в крупный шрифт. Не длинная фраза.
- Лид-магнит расписан подробно в блоке «контент» — исполнитель должен понимать что именно собирать.
- Запреты явно сформулированы.

## Production-инструкции вне стека курса (волна О, V7)

DEMO-SCREENCAST для B2B_SAAS — основной формат для софта. Стек курса (Magnific + Higgsfield) этого не делает. В брифе указывай инструмент по бюджетному режиму:
- LITE — Loom (бесплатный, до 5 минут)
- STANDARD — Screen Studio ($89 one-time, Mac) — даёт авто-зум на клики, плавный курсор
- PRO — Tella ($25/мес, мульти-сцены, ребрендинг) или Riverside.fm для интервью со студийным звуком
Постпродакшн — CapCut / Descript / Premiere.

Длительность по площадке:
- Meta CAROUSEL для B2B (статья / industry report формат) — 60-90 сек чтение
- Meta Feed/Stories 30 сек
- Reels 15 сек

Структура DEMO-SCREENCAST: Hook (3-5 сек) → Demo действия в UI (15-50 сек) → Result с цифрой (5-15 сек) → CTA «Запросите demo / Получите ROI».

Премиум-static для M&A / LEGAL / FINANCE-консалт. Кириллица в Instagram Stories editor не работает (только sans-serif шрифты). Workflow:
- Шрифты: серифные — Garamond / Tinos / EB Garamond / Lora / Source Serif Pro
- Инструменты: Photoshop / Affinity Photo (статика), InDesign / Affinity Publisher (multi-page PDF / Meta CAROUSEL для B2B документального формата)
- Палитра: navy (#0A2540) / charcoal (#1C1E26) / золотой акцент (#C9A961) / off-white фон
- Композиция: воздух, минимум элементов, фокус на одной цифре или одной фразе
- НЕ использовать Instagram editor / CapCut Studio для этого профиля

## Чего не делаешь

- Не пишешь бриф без сегмента и оффера на входе — это шум.
- Не оставляешь поле «референсы» пустым — без референсов исполнитель не попадёт в стиль.
- Не сваливаешь в одну таблицу 5 крео — каждое крео отдельной таблицей.
- Не превращаешь бриф в эссе на 3 страницы — таблица должна читаться за 60 секунд.

## Когда не работаешь

При написании текста крео — это ru-copywriter.

При генерации офферов — это offer-generator.

При проверке готового крео на политику Meta — это meta-policy-checker.

При финальной проверке артефакта перед отправкой — это quality-gate.
